# Support

## Getting help

Use GitHub Issues for:

- reproducible bugs
- documentation problems
- installation/bootstrap issues
- workflow regressions

Before filing an issue:

1. Read `README.md`
2. Check `docs/system/index.md`
3. Review `RELEASING.md` for release/pipeline questions
4. Include the exact command, environment, and error output you saw

## What to include

- operating system
- Node.js version
- npm version
- the package or workflow you were using
- relevant logs or screenshots
- steps to reproduce

## Security issues

For vulnerabilities or sensitive security concerns, do **not** use public issues. Follow `SECURITY.md`.
