# Spec Type Templates

Choose the template closest to your work:

| Type | When to use | Template file |
|------|-------------|---------------|
| \eature\ | A new product capability, UI component, or user-facing behavior | \eature.md\ |
| \workflow\ | Cross-cutting governance, pipeline, or tooling changes spanning multiple artifact types | \workflow.md\ |
| \contract\ | API, interface, schema, or protocol boundary | \contract.md\ |
| \skill\ | A new or modified AI agent skill | \skill.md\ |
| \gent\ | A scoped change to a single lane agent or subagent prompt | \gent.md\ |
| \migration\ | Data migration, schema migration, or state transition | \migration.md\ |

Copy the relevant template to \docs/specs/<your-slug>/spec.md\ and fill in the sections. Not all recommended sections are required — the spec validator only enforces the 8 required headings. Add extra sections as needed.
