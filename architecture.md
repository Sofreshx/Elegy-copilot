# Architecture Map

Purpose: describe how this repository's agentic system is organized and how it should operate inside host codebases.

## Goals
- Central source of truth for agent instructions.
- Installable into existing repositories with minimal friction.
- Auto-tailor agents/contexts to the host's tech stack.
- Surface inconsistencies early via `warnings.md`.
- Self-improve through failure analysis and instruction drift detection.

## System Components

### Kernel
- `.github/copilot.instructions.md` - Routes requests (task pipeline or free-form) to appropriate agents.

### Task Pipeline
- `raw.tasks.md` - Inbox for unrefined ideas
- `tasks.md` - Structured backlog with IDs, priorities, agents, modes
- `failed.tasks.md` - Post-mortems with failure analysis

### Health & Evolution
- `warnings.md` - Inconsistencies, risks, technical debt
- `docs/instruction-changelog.md` - Version history of instruction changes

### Meta-Agents (System Management)
| Agent | Purpose |
|-------|---------|
| `onboarding.agent.md` | Scan repo, detect stack, generate tailored agents/contexts |
| `task-creator.agent.md` | Convert raw tasks to structured tasks |
| `task-priority-planner.agent.md` | Order, batch, and track task state |
| `task-runner.agent.md` | Execute tasks with domain agents |
| `instruction-editor.agent.md` | Update instructions with feedback loop |
| `instruction-drift.agent.md` | Detect instruction decay, propose fixes |

## How to Extend
1. Add a new Context in `contexts/` describing the domain.
2. Add a Domain Agent in `agents/` using the Agent Template Schema (see `onboarding.agent.md`).
3. Update this file with the new agent/context.
4. Add to Stack Detection Matrix in `onboarding.agent.md` if auto-detection needed.
5. Add warnings for known gaps in `warnings.md`.
