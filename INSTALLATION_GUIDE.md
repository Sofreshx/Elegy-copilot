# Installation & Onboarding Guide

This guide explains how to set up the **Instruction Engine** for your projects.

## 🏗️ The Workspace Model
Unlike traditional templates, you do not copy this repository into your project. Instead, you treat it as a **Shared Library** in your VS Code Workspace.

**Structure:**
```text
MyWorkspace.code-workspace
├── my-project/           <-- Your actual code
└── instruction-engine/   <-- This repository (The Brain)
```

This allows you to update the engine (via `git pull`) without breaking your project-specific customizations.

## Installation Steps

### 1. Add to Workspace
1.  Open your project in VS Code.
2.  **File** > **Add Folder to Workspace...**
3.  Select the `instruction-engine` folder.
4.  Save the workspace file (`File` > `Save Workspace As...`) for convenience.

### 2. Initialize the Project
You need to create the local configuration folder (`.instructions/`) in your project.

**Prompt Copilot:**
> "Run the onboarding agent. Initialize this project for the Instruction Engine. Create the .instructions folder and scan the codebase for patterns."

**What happens:**
1.  **Creation**: The agent creates `.instructions/` and `.instructions-output/` in your project root.
2.  **Analysis**: It scans your code to detect stacks (React, Python, AWS, etc.).
3.  **Generation**: It creates tailored context files:
    - `.instructions/project.patterns.md`
    - `.instructions/architecture.md`
    - `.instructions/tasks.md`

### 3. Usage
Now you can use the agents defined in `instruction-engine`.

- **@planner**: "Create a plan to add a login page." (Writes to `.instructions/tasks.md`)
- **@runner**: "Implement the login page." (Reads from `.instructions/tasks.md`; defaults to batching up to 3 pending tasks unless `Mode` is set)
- **@skill-builder**: "Create a skill for this library." (Saves to `.instructions/skills/`)

## 📂 Folder Structure

| Folder | Location | Purpose |
| :--- | :--- | :--- |
| **Global Engine** | `instruction-engine/.github/` | Shared agents, generic skills, and templates. **Do not edit** (unless contributing to the engine). |
| **Local Config** | `my-project/.instructions/` | Your project's specific rules, tasks, and custom skills. **Edit this.** |
| **Local Output** | `my-project/.instructions-output/` | Reports, logs, and debug info generated by agents. |

## 🔄 Customizing Skills
If you need to override a global skill (e.g., you have a specific way of doing `react`):
1.  Create `.instructions/skills/react.skill.md`.
2.  The Kernel will automatically prioritize your local file over the global `instruction-engine/.../react.skill.md`.
